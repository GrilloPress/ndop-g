Design hypotheses

November-December

In lab 6, users found navigating between pages easier but were unsure which page contained the information they wished to see.

If we update the page headers to reflect the subjects in the users' own voice
then we will improve navigation
because users will easily be able to find further information that is relevant to them

In lab 6, users didn't use the bottom of the page pagination links. Some users on UserZoom used them extensively.

If we increase the size of the next/prev label and reduce the page title slightly
then users will better understand how to cycle through pages
because users will immediately understand the concept that the bottom of the page link takes them to the next page/topic.

In lab 6, users immediately scrolled past the blue banner to explore content, even when tasked with finding other pages.

If we change the blue box link header to a non-coloured box
then users will be better able to navigate between pages
because it will be easier to read the links and users won't think that it is just a graphical element

In lab 6, one user clicked a link but turned their head. When they looked back they didn't see the page load and thought the page had failed to change state

If we change the blue box link header to a non-coloured box
then users will notice the change between pages
because the page change will be indicated by link colour and state change, compared to just text-decoration change

In lab 6, users didn't understand the scope of the service after interacting with the landing page

If we summarise content a user needs to understand on the landing page
then users will understand the service better
because they will be able to scan the homepage and capture the main features of the service

If we display the summary content in a visually appealing way
then users will understand the service better
because as studies show, they will be spend more time engaging with the content directly related to it
